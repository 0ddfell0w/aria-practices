Menu Button Widget

Keyboard Interaction and Focus Management

Key Function
Space or Enter Key Open menu and move focus to first menuitem
Up Arrow Key Open menu and move focus to previous menuitem
Down Arrow Key Open menu and move focus to next menuitem

ARIA Roles, Properties, and States

Role Property/State Element Usage
button a
  • The role=button is added to the a element to identify the element as an ARIA button, overriding the default role of link.
href a
  • [#]: The use of the href="#" attibute makes the a element part of the tab ordering of the page.
aria-controls a
  • [IDREF]: Is a pointer to the popup menu, by using the id of the element with role=menu.
aria-haspopup a
  • [true]: Defines an element with role=button as a ARIA menu button design pattern.
aria-expanded a
  • [false]: Indicates the popup menu is open.
  • [true]: Indicates the popup menu is closed.

Popup Menu Widget

Keyboard Interaction and Focus Management

Key Function
Space or Enter Keys
  • Activates the link associated with the menuitem.
Up Arrow Key Move focus to previous menuitem.
Down Arrow Key Move focus to next menuitem.
Escape Key Move focus back to menu button and then close menu. NOTE: This sequence is important since hidding the element with focus will cause the focus to move to the beginning of the page.

ARIA Roles, Properties, and States

Role Property/State Element Usage
menu ul
  • Idenfities an ARIA menu.
  • Does not need a tabindex value since the element does not receive keyboard focus.
aria-labelledby ul
  • [IDREF]: Pointer to element with the id that defines the accessible name for the menu, in this example "WAI-ARIA Quick Links".
menuitem a
  • Identifies element as an ARIA menuitem.
tabindex a
  • -1: Identifies each li[role=menuitem] as an interactive element on the page (e.g. can recived keyboard focus).
