Navigation Menubar Example: Actions

NOTE: Please provide feedback on this example in issue 144.

Following is the implementations of the design pattern for menubar that demonstrate how a menubar can be used for actions. Each item in the menubars represents a CSS styling and opens pull down menu items that change the styling of the text. This example also demonstrates the usage of menuitemradio and menuitemcheckbox.

Menubar Widget Features

Keyboard Support

Key Function
Space or Enter Key
  • Open or close popup menu.
  • Focuse stays on menubar menu item.
Left Arrow Key
  • Moves focus to previous menubar menu item.
  • If focus on first menubar menu item, focus is moved to last menubar menu item.
Right Arrow Key
  • Moves focus to next menubar menu item.
  • If focus on last menubar menu item, focus is moved to first menubar menu item.
Down Arrow Key
  • Open popup menu.
  • Moves focus to first popup menu item.
Up Arrow Key
  • Open popup menu.
  • Moves focus to last popup menu item.
Home
  • Moves focus to first menubar menu item.
End
  • Moves focus to last menubar menu item.
Character Key
  • Moves focus to next menubar item that starts with that letter.
  • If no menubar menu item starts with the letter, focus does not move.

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
menubar ul
  • Identifies a ARIA menubar widget.
  • No tabindex value is needed since the role="menubar" does not receive focus.
  • Accessible name for the menu is defined by the aria-label attribute.
aria-label="string" ul
  • Defines an acessible name for the menu bar.
  • Used to help people understand the purpose of the menubar and uniquely identify the menubar from other menubars on the page.
menuitem span
  • Identifies a ARIA menuitem in the menubar widget.
  • Accessible name for the menuitem is defined by the text content of the span element.
tabindex="-1" span
  • Makes the span element keyboard focusable, but not part of tab order of the page.
tabindex="0" span
  • Makes the span element keyboard focusable and part of tab order of the page.
  • Only one menubar item should have tabindex="0".
  • The default menubar item with tabindex="0" is the first menubar item (e.g page is loaded).
  • The last menubar item to have focus is given the tabindex="0" so that when user tabs through the page focus will be given to the last menubar item that the user used.
aria-haspopup="true" span
  • Identifies the menuitem as a controller for a popup menu.
  • The menuitem is only to open and close the popup menu it cannot be a link.
aria-expanded="true" span
  • Indicates popup menu is open.
aria-expanded="false" span
  • Indicates popup menu is closed.

Role, Property, State, and Tabindex Attributes

Role Attribute Element Usage
menuitem span
  • Identifies a ARIA menuitem in the menu widget.
  • Accessible name for the menuitem is defined by the text content of the span element.
tabindex="-1" span
  • Makes the span element keyboard focusable, but not part of tab order of the page.
aria-haspopup="true" span
  • Identifies the menuitem as a controller for a popup menu.
  • The menuitem is only to open and close the popup menu it cannot be a link.
aria-expanded="true" span
  • Indicates popup menu is open.
aria-expanded="false" span
  • Indicates popup menu is closed.
menu ul
  • Identifies a ARIA menu widget.
  • No tabindex value is needed since the role="menu" does not receive keyboard focus.
  • Accessible name for the menu is defined by the aria-label attribute.
menuitemradio li
  • menuitemradio is a checkable menuitem in a set of elements with the same role, only one of which can be checked at a time.
  • Authors must ensure that multiple menuitemradios are owned by an element with role group, menu, or menubar in order to identify that they are related widgets.(Note: menuitemradio in Font-family is owned by menu role in the example.)
  • If a menu or menubar contains more than one group of menuitemradio elements, or if the menu contains one group and other, unrelated menu items, authors should nest each set of related menuitemradio elements in an element using the group role, and authors should delimit the group from other menu items with an element using the separator role.(Note: Style/Color menu bar menu item has two group roles of text color and text decoration with separator role in the example.)
aria-checked="true" (or "false") li
  • Indicates popup menu item is checked (or unchecked.)
group ul
  • Used for grouping of menuitemradio popup menu items.
aria-label="string" ul
  • Defines an acessible name for menuitemradio group.
  • Used to help people understand the name of menuitemradio group which distinguish from other menuitemradio group.
menuitemcheckbox li
  • menuitemcheckbox is a menuitem with a checkable state whose possible values are true, false, or mixed.
  • Authors must ensure that menu item checkboxes are owned by an element with role menu or menubar in order to identify that they are related widgets. In this example, menuitemcheckbox is owned by role menu.
aria-checked="true" (or "false") li
  • Indicates popup menu item is checked (or unchecked.)
